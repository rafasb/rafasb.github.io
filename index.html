Aprendiendo juntos

con Rafael Santisteban

  • 01

    PiHole en docker


    Desplegar PiHole usando docker y docker-compose.

    Muy útil tanto para desplegar en casa como para realizar las primeras prácticas con docker y docker-compose.

    Permite disponer de un sistema para bloquear publicidad en los dispositivos de casa y un servicio de DNS propio para incorporar tus dispositivos.

  • 02

    Wordpress en docker con certificado Let's Encrypt


    Útil para poner rápidamente en marcha un proyecto wordpress y poder mostrarlo públicamente con seguridad SSL.

    Hace uso de un proxy inverso para facilitar el acceso mediante cualquier nombre DNS y disponer del certificado gratuido y actualizado del servicio de Let's Encrypt.

    Al emplear docker facilita el backup y el cambio de proyecto simplemente copiando y restaurando los directorios de datos.

  • 03

    Wordpress en docker con traefik y certificado Let's Encrypt


    Útil para poner rápidamente en marcha un proyecto wordpress y poder mostrarlo públicamente con seguridad SSL.

    Hace uso de traefik que además de aportar un balanceado de carga y certificado SSL automatizado, nos muestra estadísticas de uso.

    Es un excelente ejemplo para configurar traefik, ampliamente empleado con docker y kubernetes.

  • 04

    Gitlab sobre docker-compose


    Despliegue local de la potente aplicación GitLab sobre docker-compose.

    Servidor Git para control de versiones, servicio de CI/CD, registro de imágenes de contenedores

    También es interesante el script comentado gen-certs.sh para gestión de certificados.

  • 05

    Monitorización de logs


    Despliegue local de Graylog, basándose en elasticsearch y fluentd para recopilar origenes de logs.

    Complentando la gran utilidad de Graylog para realizar análisis de logs, generar alertas y dashboards, disponemos de Fluentd para recopilar y realizar un primer tratamiento de los logs.

    Es una solución muy potente y escalable donde combina diferentes aplicaciones vinculadas entre si.

  • 06

    Monitorización de métricas


    Despliegue local de Grafana, basándose en Influxdb + Telegraf y Prometheus.

    Veremos dos aproximaciones diferentes para la monitorización de sistemas, una basada en Prometheus y otra basada en InfluxDB v2 y Telegraf.

    Este es un ejemplo de un conjunto complejo de contenedores que realiza muy diferentes funciones para conseguir un resultado, disponer de paneles para monitorizar métricas de sistemas.

  • 07

    Contenedor para ejecutar Ansible


    Ansible es una de las mejores soluciones para automatización. Este proyecto lo convierte en una aplicación portable

    El proyecto explora la posibilidad de crear nuestro propio contenedor, usando una imagen de Ubuntu e instalando aplicaciones de sistema, aplicaciones Python e incluso plugins y configuración de la aplicación.

    El resultado es una imagen que podemos usar para ejecutar un contenedor como un comando CLI para realizar tareas automatizadas.

  • 08

    Contenedor para ejecutar Jupiter


    Combina la potencia de Ansible con la facilidad de uso y practicidad de Jupyter Notebook

    El proyecto crea un servidor de Jupyter Notebook (última versión Jupiter-Lab).

    Permite emplear la potente herramienta de Ansible, con una colección de plugins orientada a la automatización de equipos de red y a través de la interfaz web de Jupyter
